# Cloudtype "오류" / "중지됨" 해결 방법

## 화면이 말해 주는 것

### 1. 상태: **중지됨** (빨간 점)
- 서비스가 **실행 중이 아님** → 배포가 성공해도 "오류" 때문에 계속 중지된 상태일 수 있음.

### 2. 배포 내역: 전부 **오류** 또는 **취소됨**
- "Add Cloudtype deployment guides", "Initial commit: Express + MongoDB Atlas todo API" 등으로 배포를 시도했지만 **매번 실패**한 상태.

### 3. 플랫폼 표시: **next.js**
- 이 프로젝트는 **Next.js가 아니라 Express(Node.js)** 백엔드입니다.  
- Cloudtype이 Next.js로 인식하면 **빌드/시작 명령**이 맞지 않아 오류가 납니다.

---

## 해결 순서

### 1. 설정에서 "플랫폼/빌드" 수정

1. **설정** 탭 클릭.
2. **빌드 설정** 또는 **Build / Start** 관련 항목 찾기.
3. 아래처럼 맞추기:
   - **프레임워크/플랫폼**: `Node.js` (Next.js 아님)
   - **빌드 명령**: `npm ci` 또는 `npm install`
   - **시작 명령**: `npm start`
4. **저장** 후 **다시 배포**(재배포) 실행.

### 2. 환경 변수 확인

- **설정** → **환경 변수** (또는 시크릿):
  - **MONGO_URL** = MongoDB Atlas 연결 문자열 전체  
    (로컬 `.env`에 있는 값 그대로, **시크릿**으로 저장 권장)

### 3. 배포 후 확인

- 배포 내역에서 **초록색(성공)** 이 나오는지 확인.
- 성공 후 **접속하기** / **연결** 에 나오는 URL로 접속:
  - `https://해당URL/` → "서버가 동작 중입니다."
  - `https://해당URL/api/db-info` → Atlas 연결 확인.

---

## 요약

| 문제 | 원인 | 조치 |
|------|------|------|
| 중지됨 | 배포 실패로 서비스가 안 뜸 | 빌드/시작 설정 수정 후 재배포 |
| 오류 | Next.js로 인식해 잘못된 빌드/시작 | Node.js + `npm start` 로 변경 |
| MONGO 연결 실패 가능 | 환경 변수 없음/오타 | MONGO_URL 시크릿으로 설정 |

**핵심:** **Node.js + 시작 명령 `npm start`** 로 바꾼 뒤 저장하고, **한 번 더 배포**하면 됩니다.

## (권장) 서비스 새로 만들기

설정에서 플랫폼을 바꿀 수 없다면:

1. **기존 서비스 삭제** (vibe-todo-backend)
2. **새 서비스 생성** → `⌘ + K` 또는 시작하기에서 **node** 검색
3. **Node.js** 템플릿 선택 (Next.js 아님)
4. **저장소 선택**: `editor-7/vibe_todo_backend` · main
5. **환경 변수** 에 **MONGO_URL** (시크릿) 설정
6. **배포하기** 클릭

저장소에 `.cloudtype/app.yaml` 이 있으면 Node.js로 빌드/시작됩니다.
